# Proposal: OpenSpec Presentation for Java Developers

## Why

Java developers are often bogged down by complex architectures and "vibe coding" (ad-hoc AI prompting) which leads to inconsistent results. OpenSpec provides a structured, spec-driven development (SDD) workflow that brings predictability, maintainability, and clear intent to AI-assisted coding. This presentation aims to introduce these concepts in an engaging, visual way that resonates with the Java community.

## What Changes

We will build a comprehensive Reveal.js presentation located in `presentations/openspec/`. The presentation will cover:
- The shift from "Vibe Coding" to "Spec-Driven Development".
- How OpenSpec acts as a bridge between human intent and AI execution.
- Real-world application of OpenSpec in a Java context.
- High-quality visuals, Mermaid diagrams for architectural concepts, and relatable MEMEs for audience engagement.

## Capabilities

### New Capabilities
- `intro-to-sdd`: Introduction to Spec-Driven Development vs. traditional development.
- `openspec-workflow`: Detailed breakdown of the OpenSpec artifacts (proposal, design, specs, tasks).
- `java-integration`: Specific examples of how OpenSpec handles Java-specific concerns like type safety and null handling.
- `visual-engagement`: A library of Mermaid diagrams and curated MEMEs to be used throughout the presentation.

### Modified Capabilities
- `presentation-framework`: Update the existing landing page and sidebar to include the new OpenSpec talk.

## Impact

- `presentations/openspec/index.md`: Main slide content.
- `presentations/openspec/images/`: Assets directory for MEMEs and custom graphics.
- `index.html`: Project landing page updated with the new talk link.
- `AGENTS.md`: Updated to include instructions for the OpenSpec presentation.
